# Django Migrations Inspector 0.1.0

`0.1.0` is the first public beta release of Django Migrations Inspector.

This release introduces a local-first migration safety toolkit for Django teams that want clearer answers before deploys, during CI, and when planning rollback paths. It focuses on readable, deterministic reports about migration graph health, pending migration risk, historical migration review, and rollback impact without requiring a hosted service.

## Highlights

- Migration graph inspection with visibility into multiple heads, merge migrations, roots, leaves, and dependency hotspots.
- Pending migration risk analysis focused on destructive schema changes, irreversible data code, raw SQL, unknown operations, and other review-needed steps.
- Historical migration audit for reviewing migration files already in the repository, even when nothing is currently pending.
- Rollback simulation with blockers, blast radius, cross-app impact, and reverse-step previews.
- Deterministic text, JSON, Mermaid, and Graphviz DOT output for both direct CLI use and automation.

## Compatibility

- Python 3.10+
- Django 4.2 through 6.x

## Release notes

This is a beta release. The command surface is ready for real project use, while the project is still early enough to keep improving its risk rules, rollout guidance, and migration heuristics over the next releases.

For installation, usage, and documentation, see the project README and docs site.
